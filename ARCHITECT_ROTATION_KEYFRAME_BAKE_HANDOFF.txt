RotoMotion Architect Handoff: Rotation Keyframes Visible, But Not Baking
=======================================================================

Context
-------

User-visible problem:
- Rotation keyframe ticks are visible in the timeline/viewport.
- Therefore the selected-joint rotation key data exists in app state.
- Bake Animation still does not appear to bake those keyed rotations into the exported animation.

High-level expected flow:

1. User edits selected joint Euler XYZ.
2. User clicks Add Rotation Key.
3. Key is stored in rotationOverrideLayer.keyframesByJoint[selectedJoint].
4. Bake Animation loops all solved frames.
5. For each solved frame:
   - rebuild curve-pinned pose
   - evaluate rotation override for that joint at that frame
   - replace bone.simdEulerAngles with the override
   - sample bone local transforms into BakedRigAnimation
6. Export writes bakedRigAnimation.asSessionArmaturePoseBuffer() to session_armature_pose.json.
7. Python USD writer reads localRotationEulerXYZ / rotation_euler_xyz from that JSON and converts Euler XYZ to USD quaternions privately.

Important: If the timeline tick is visible, do not debug key creation first. The most likely bug is between bake evaluation, bone application, sampled baked data, and export JSON.

```swift
import Foundation

struct JointRotationOverrideLayer: Codable {
    let schema: String
    var selectedJoint: String
    var cleanKeysEnabled: Bool
    var keyframesByJoint: [String: [Keyframe]]

    struct Keyframe: Codable, Identifiable {
        var id: String { "\(frameIndex)" }

        let frameIndex: Int
        let timeSeconds: Double

        /// Absolute replacement local Euler XYZ radians.
        var eulerXYZ: [Double]

        enum CodingKeys: String, CodingKey {
            case frameIndex
            case timeSeconds
            case eulerXYZ
            case axisValuesXYZ
        }

        init(
            frameIndex: Int,
            timeSeconds: Double,
            eulerXYZ: [Double]
        ) {
            self.frameIndex = frameIndex
            self.timeSeconds = timeSeconds
            self.eulerXYZ = eulerXYZ
        }

        init(from decoder: Decoder) throws {
            let container = try decoder.container(keyedBy: CodingKeys.self)

            frameIndex = try container.decode(Int.self, forKey: .frameIndex)
            timeSeconds = try container.decode(Double.self, forKey: .timeSeconds)

            if let euler = try container.decodeIfPresent([Double].self, forKey: .eulerXYZ),
               euler.count == 3 {
                eulerXYZ = euler
            } else if let legacyAxis = try container.decodeIfPresent([Double].self, forKey: .axisValuesXYZ),
                      legacyAxis.count == 3 {
                eulerXYZ = legacyAxis
            } else {
                eulerXYZ = [0, 0, 0]
            }
        }

        func encode(to encoder: Encoder) throws {
            var container = encoder.container(keyedBy: CodingKeys.self)
            try container.encode(frameIndex, forKey: .frameIndex)
            try container.encode(timeSeconds, forKey: .timeSeconds)
            try container.encode(eulerXYZ, forKey: .eulerXYZ)
        }
    }

    static let `default` = JointRotationOverrideLayer(
        schema: "com.gravitas.rotomotion.rotation_override_layer.v0",
        selectedJoint: "Head",
        cleanKeysEnabled: false,
        keyframesByJoint: [:]
    )
}
```

Canonical Override Evaluator Currently In ViewModel
---------------------------------------------------

File:
Gravitas RotoMotion/Gravitas RotoMotion/UI/RotoMotionViewModel.swift

Current code:

```swift
func rotationOverrideEulerForFrame(
    joint: String,
    frameIndex: Int,
    timeSeconds: Double
) -> SIMD3<Float>? {
    _ = timeSeconds

    let keys = (rotationOverrideLayer.keyframesByJoint[joint] ?? [])
        .sorted { $0.frameIndex < $1.frameIndex }

    if !keys.isEmpty {
        if let exact = keys.first(where: { $0.frameIndex == frameIndex }) {
            return eulerFromRotationKey(joint: joint, key: exact)
        }

        if let previous = keys.last(where: { $0.frameIndex <= frameIndex }) {
            return eulerFromRotationKey(joint: joint, key: previous)
        }

        return eulerFromRotationKey(joint: joint, key: keys[0])
    }

    if let live = liveRotationOverrideEulerXYZByJoint[joint] {
        return ManualRotationConstraint.clampedEulerXYZ(
            joint: joint,
            values: live
        )
    }

    if let held = heldRotationOverrideEulerXYZByJoint[joint] {
        return ManualRotationConstraint.clampedEulerXYZ(
            joint: joint,
            values: held
        )
    }

    return nil
}

private func eulerFromRotationKey(
    joint: String,
    key: JointRotationOverrideLayer.Keyframe
) -> SIMD3<Float>? {
    guard key.eulerXYZ.count == 3 else {
        return nil
    }

    return ManualRotationConstraint.clampedEulerXYZ(
        joint: joint,
        values: SIMD3<Float>(
            Float(key.eulerXYZ[0]),
            Float(key.eulerXYZ[1]),
            Float(key.eulerXYZ[2])
        )
    )
}
```

Rules:
- If keys exist for a joint, keys drive that joint.
- If no keys exist, live/held override drives that joint.
- Evaluation is frameIndex based. timeSeconds is ignored.

Possible failure point:
- UI key frameIndex might not match solve frameIndex.
- Previous-key fallback should still produce stepped animation if solve frameIndex increases through the same index domain.
- If solve frameIndex values do not overlap UI currentFrameIndex values at all, all bake frames may use first key only.

```swift
    frameIndex: Int?,
    timeSeconds: Double,

    liveRotationOverrideEulerXYZByJoint: [String: SIMD3<Float>],

        return ManualRotationConstraint.clampedEulerXYZ(
            joint: joint,
            values: live
        )
    }

    if let held = heldRotationOverrideEulerXYZByJoint[joint] {
        return ManualRotationConstraint.clampedEulerXYZ(
            joint: joint,
            values: held
```

Likely failure points in this path:

1. Joint name mismatch:
   - Key list can contain selectedJoint, but bake only applies if session.bonesByCanonicalName[joint] exists.
   - If selectedRotationJoint is not a key in bonesByCanonicalName, bake silently skips it.
   - If selected joint is not in session.jointOrder, BakedRigAnimationSampler also will not sample it.

2. Frame index mismatch:
   - Keys are saved using currentFrameIndex.
   - Bake evaluates using solvedFrame.frameIndex.
   - If those index domains differ, exact matching fails.
   - Previous-key fallback should produce stepped behavior only if solvedFrame.frameIndex is monotonically in the same range.

3. The override is applied, but sample does not prove it:
   - Need log the selected joint before override, evaluator output, after override bone.simdEulerAngles, and sampled BakedRigAnimation value.

4. Stale exported JSON:
   - Export writes from bakedRigAnimation. If user does not rebake after edits, export refuses only if bakedRigAnimation was nilled.
   - Confirm bakedRigAnimation is nil after every edit in the actual running build.

5. SceneKit Euler normalization:
   - bone.simdEulerAngles may normalize or represent rotations differently after assignment.
   - Still, if euler assignment changes visual pose, the sampled Euler should differ.

```swift
enum BakedRigAnimationSampler {
    static func sample(
        session: SkinnedRigSession,
        frameIndex: Int,
        timeSeconds: Double,
        jointNames: [String]
    ) -> BakedRigAnimation.Frame {
        var joints: [String: BakedRigAnimation.JointTransform] = [:]

        for joint in jointNames {
            guard let bone = session.bonesByCanonicalName[joint] else {
                continue
            }

            let p = bone.simdPosition
            let euler = bone.simdEulerAngles
            let s = bone.simdScale

            joints[joint] = .init(
                localTranslationXYZ: [
                    Double(p.x),
                    Double(p.y),
                    Double(p.z)
                ],
                localRotationEulerXYZ: [
                    Double(euler.x),
                    Double(euler.y),
                    Double(euler.z)
                ],
                localScaleXYZ: [
                    Double(s.x),
                    Double(s.y),
                    Double(s.z)
                ]
            )
        }

        return .init(
            frameIndex: frameIndex,
            timeSeconds: timeSeconds,
            joints: joints
        )
    }
}
```

```swift
enum SessionArmaturePoseBufferJSONExporter {
    static func write(
        buffer: SessionArmaturePoseBuffer,
        to url: URL
    ) throws {
        var joints: [String: [[String: Any]]] = [:]

        for jointName in CanonicalRig.jointNames {
            let keys = buffer.frames.compactMap { frame -> [String: Any]? in
                guard let transform = frame.joints[jointName] else {
                    return nil

                    "time": frame.timeSeconds,
                    "localTranslationXYZ": transform.localTranslationXYZ,

                    "localScaleXYZ": transform.localScaleXYZ,

                    "rotation_euler_xyz": transform.localRotationEulerXYZ,
                    "scale_xyz": transform.localScaleXYZ,
                    "curve": "linear"
                ]
            }

            if !keys.isEmpty {
                joints[jointName] = keys
            }
        }

        let root: [String: Any] = [
            "schema": buffer.schema,
            "clipID": buffer.clipID,
            "frameCount": buffer.frames.count,
            "fps": buffer.fps,
            "sourceKind": "posed_armature_local_transforms",
            "transform_space": "local_parent",
            "rotation_order": "euler_xyz_radians",
            "translation_policy": "all_local",
            "scale_policy": "all_local",
            "joints": joints
        ]
    }
}
```

```python
    scales = list(rest_scales)

    for canonical_name, target_index in joint_map.items():
        key = sample_joint_at_frame(solved_joints[canonical_name], frame)
        rotation = quat_from_euler_xyz(*key["rotation_euler_xyz"])

        if direct_local_transforms:
            translations[target_index] = vec3f(*key["translation_xyz"])
            rotations[target_index] = rotation
            scales[target_index] = Gf.Vec3h(...)
        else:
            rotations[target_index] = rest_rotations[target_index] * rotation
```

Most Likely Root Causes
-----------------------

Based on the current code, the likely causes are:

1. Frame index domain mismatch:
   - UI stores keys at currentFrameIndex.
   - Bake evaluates using solvedFrame.frameIndex.
   - Need prove these are the same timeline index.

2. Joint name mismatch:
   - Timeline uses selectedRotationJoint.
   - Bake only applies/samples if bonesByCanonicalName[selectedRotationJoint] exists and jointOrder includes it.

3. The wrong bake/export artifact is being inspected:
   - Export uses bakedRigAnimation only.
   - The generated session_armature_pose.json is the source of truth to inspect.

4. Duplicate evaluator design:
   - Viewport uses JointRotationOverrideApplier.
   - Bake uses RotoMotionViewModel.rotationOverrideEulerForFrame.
   - These should be unified before further debugging.

5. Stale bake restoration:
   - bakedRigAnimation is persisted and loaded.
   - Edits should nil it, but verify in the running app that the status changes to "Bake is stale. Re-bake animation." after every key/edit.
